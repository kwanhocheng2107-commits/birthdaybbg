# ChatGPT 项目交接说明

## 项目概览

这是一个送给女朋友的生日网页，项目名为《给你的第1个生日宇宙》。当前是手机端优先的原生 HTML + CSS + JavaScript 单页网页，不使用 React、后端、数据库或登录系统。

正式页面入口是 `index.html`，样式在 `style.css`，交互逻辑在 `script.js`。页面包含生日倒计时开场、进入生日宇宙转场、回忆时间线、互动卡片、生日签与问答小彩蛋、生日信和本地留言盒。

## 当前已完成

- 生日倒计时三状态逻辑：生日前、生日当天、生日之后。
- `DEV_UNLOCK = true` 开发测试入口。
- 入口转场已调整为“倒计时页星尘退场 -> 第二页淡入”的串行流程。
- 月亮作为全局背景图层保留，不随倒计时页消失。
- 流星效果已改为纯 CSS 的 3 条错峰流星，不再依赖正式页里的 `meteor.js`。
- 背景星空已改为 `bg.jpg`，月亮已改为 `moon.png`。
- 月亮旧 CSS 圆形光晕已移除，现在 `.moon` 只是图片容器。
- 页面文案已集中到 `script.js` 顶部的 `contentData` 对象，后续主要改这里即可。
- 留言盒使用 `localStorage`，支持保存、读取和删除留言。

## 重要约束

- 不要把项目改成 React、Vue、Next 或其他框架。
- 不要添加后端、数据库、登录或云端留言同步。
- 不要重构整个页面结构。
- 后续修改应小步迭代，一次只处理一个明确问题。
- 不要破坏入口转场：倒计时页先退场，第二页再淡入。
- 不要随意修改流星动画，除非用户明确要求。
- 不要修改 `prefers-reduced-motion` 相关处理。

## 关键文件

- `PROJECT_SPEC.md`：完整项目规格和固定要求。
- `TODO.md`：当前已完成、待优化和建议顺序。
- `CODEX_HANDOFF.md`：给 Codex 的详细交接说明。
- `index.html`：页面结构，当前文案多数已改为挂载点。
- `style.css`：视觉、布局、背景、动画。
- `script.js`：倒计时、转场、文案数据渲染、卡片、小游戏、留言盒。
- `bg.jpg`：当前星空背景图。
- `moon.png`：当前月亮图片。
- `meteor.js` / `meteor-demo.html`：旧流星调试文件，正式页目前不引用。

## 后续建议

1. 先替换 `script.js` 顶部 `contentData` 里的真实文案。
2. 再逐项检查手机端排版，尤其是标题、卡片背面、生日信段落。
3. 最后做完整移动端 QA：入口转场、滚动、流星、留言盒、本地存储、reduced-motion。

如果需要继续开发，请先阅读 `PROJECT_SPEC.md`、`TODO.md`、`CODEX_HANDOFF.md` 和本文件，再根据用户当次明确要求只改相关模块。
